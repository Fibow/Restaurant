# Система управления рестораном (REST API)

RESTful API, созданное на базе Spring Boot, для управления базовыми процессами ресторана: ведением списка заведений, составлением меню и обработкой клиентских заказов.

## Стек технологий
* **Язык:** Java 17
* **Фреймворк:** Spring Boot (Spring Web, Spring Data JPA, Validation)
* **Сборщик проекта:** Maven
* **Деплой (Облако):** Azure Web Apps

## Основные возможности
* **Управление ресторанами:** Создание, чтение, обновление и удаление (CRUD) информации о ресторанах.
* **Управление меню:** Добавление, просмотр и удаление блюд с указанием их стоимости.
* **Обработка заказов:** Создание и полное обновление заказов (реализована связь «Многие ко многим»).
* **Умная бизнес-логика:** Сервер автоматически высчитывает итоговую сумму (`totalPrice`) для каждого нового или обновленного заказа на основе выбранных пунктов меню.
* **Валидация данных:** Защита от некорректного ввода (например, блокировка пустых названий или отрицательных цен).
* **Интерактивная документация:** Встроенный веб-интерфейс для тестирования API прямо в браузере.

## Как запустить и протестировать проект

**Вариант 1: В облаке**
Проект уже развернут в Microsoft Azure и подключен к облачной базе данных. Вы можете тестировать его:
* Через интерактивную документацию: [Swagger UI](https://restaurant-api-bex.azurewebsites.net/swagger-ui/index.html)
* Или отправляя запросы через Postman на базовый URL: `https://restaurant-api-bex.azurewebsites.net`

**Вариант 2: Локально на компьютере**
1. Склонируйте этот репозиторий на свой компьютер.
2. Откройте проект в IntelliJ IDEA (или любой другой Java IDE).
3. Убедитесь, что в файле `application.properties` прописана строка подключения к базе данных PostgreSQL.
4. Запустите приложение через главный класс (`RestaurantApplication`).
5. Сервер запустится на порту `8080`, локальный Swagger будет доступен по адресу `http://localhost:8080/swagger-ui/index.html`.

## Точки доступа API (Endpoints)

### 1. Рестораны
| Метод | Эндпоинт | Описание |
|--------|----------|-------------|
| GET | `/api/restaurants` | Получить список всех ресторанов |
| POST | `/api/restaurants` | Добавить новый ресторан |
| POST | `/api/restaurants/by-params` | Добавить новый ресторан с помощью параметров запроса |
| PUT | `/api/restaurants/{id}` | Обновить данные ресторана по его ID |
| DELETE | `/api/restaurants/{id}` | Удалить ресторан по ID |

### 2. Меню (Блюда)
| Метод | Эндпоинт | Описание |
|--------|----------|-------------|
| GET | `/api/menu-items` | Получить список всех блюд |
| GET | `/api/menu-items/{id}` | Получить конкретное блюдо по его ID |
| POST | `/api/menu-items` | Добавить новое блюдо в базу |
| DELETE | `/api/menu-items/{id}` | Удалить блюдо из меню по ID |

### 3. Заказы
| Метод | Эндпоинт | Описание |
|--------|----------|-------------|
| GET | `/api/orders` | Получить список всех заказов |
| GET | `/api/orders/{id}` | Получить конкретный заказ по его ID |
| POST | `/api/orders` | Создать новый заказ (сумма рассчитается автоматически) |
| PUT | `/api/orders/{id}` | Полностью обновить заказ и пересчитать сумму |
| DELETE | `/api/orders/{id}` | Удалить заказ по ID |
